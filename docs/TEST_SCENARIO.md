## 1. 테스트 환경 준비

프로젝트 루트 디렉터리(`/Users/kangsuek/pythonProject/coworker`)에서 실행합니다.

```bash
# 1. 의존성 설치 (최초 1회 혹은 패키지 변경 시)
just setup

# 2. 데이터베이스 초기화 및 마이그레이션
just migrate

# 3. 개발 서버 실행 (백엔드 + 프론트엔드 동시)
just dev
```

- 프론트엔드: `http://localhost:5173`
- 백엔드: `http://localhost:8000`
- Swagger UI: `http://localhost:8000/docs`

> **사전 조건**: Claude CLI가 설치되어 있고 `claude -p "hello"` 명령이 정상 응답해야 합니다.

## 2. 시나리오 1: 레이아웃 및 UI 구조

**목적**: PRD Section 10.1 레이아웃 명세 준수 확인
**관련 요구사항**: F-020, F-030, F-032, F-033, F-040

### 절차

1. 브라우저에서 `http://localhost:5173`에 접속합니다.
2. 화면 구조를 확인합니다.

### 예상 결과

- [ ] **3패널 레이아웃**: 좌측 사이드바(세션 목록) + 중앙(User Channel) + 우측(Agent Channel)이 나란히 표시됩니다.
- [ ] **사이드바**: 좌측에 `w-64` 너비의 세션 목록 영역이 표시됩니다. 상단에 "새 세션" 버튼(+ 새 세션)이 존재합니다.
- [ ] **User Channel 헤더**: ☰ 버튼 + 세션 제목("새 대화")이 표시됩니다.
- [ ] **User Channel 본문**: 빈 상태에서 "무엇이든 물어보세요" 안내 메시지가 표시됩니다.
- [ ] **User Channel 입력 영역**: 하단에 텍스트 입력란 + "전송" 버튼이 존재합니다.
- [ ] **Agent Channel 헤더**: "Agent Channel" 텍스트가 표시됩니다. 메시지가 없으므로 내보내기 버튼은 숨겨져 있습니다.
- [ ] **Agent Channel 대기 상태**: 🤖 아이콘 + "대기 중" + "Team 작업이 시작되면 Agent 간 대화가 표시됩니다" 메시지가 표시됩니다 (F-033).
- [ ] **Agent Channel 읽기 전용**: Agent Channel에 텍스트 입력란이나 전송 버튼이 없습니다 (F-032).

## 3. 시나리오 2: Solo 모드 기본 흐름

**목적**: 단순 질문 → Solo 분류 → Reader 직접 응답 전체 흐름 확인
**관련 요구사항**: F-001, F-002, F-005, F-020, F-033

### 절차

1. User Channel 입력란에 `안녕하세요! 당신은 누구인가요?`를 입력합니다.
2. `Enter` 키를 누르거나 "전송" 버튼을 클릭합니다.
3. 상태 변화를 관찰합니다.

### 예상 결과

- [ ] **사용자 메시지 표시**: 입력 즉시 파란색 말풍선(우측 정렬)으로 메시지가 나타납니다 (F-020).
- [ ] **입력 비활성화**: 메시지 전송 후 입력란이 비활성화(회색 배경)됩니다.
- [ ] **상태 배지 — thinking**: "분석 중..." 상태 배지가 애니메이션(pulse)과 함께 표시됩니다 (F-005).
- [ ] **상태 배지 — solo**: "응답 생성 중..." 상태로 변경됩니다 (F-005).
- [ ] **Reader 응답**: 완성된 응답이 회색 말풍선(좌측 정렬)으로 표시됩니다 (F-002).
- [ ] **상태 배지 소멸**: 응답 완료 후 상태 배지가 사라집니다.
- [ ] **입력 활성화**: 응답 완료 후 입력란이 다시 활성화됩니다.
- [ ] **자동 스크롤**: 응답이 나타날 때 최하단으로 자동 스크롤됩니다.
- [ ] **Agent Channel 유지**: 우측 Agent Channel은 여전히 "대기 중" 상태입니다 (F-033).
- [ ] **성능**: 전체 응답이 15초 이내에 완료됩니다.

## 4. 시나리오 3: 마크다운 및 코드 구문 강조

**목적**: Reader 응답의 마크다운 렌더링 + 코드 하이라이팅 확인
**관련 요구사항**: F-021

### 절차

1. 동일한 채팅창에 다음을 입력하고 전송합니다:
   ```
   파이썬으로 피보나치 수열을 계산하는 함수를 작성해줘. 코드에 대해 마크다운 리스트로 설명도 해줘.
   ```
2. Reader 응답을 확인합니다.

### 예상 결과

- [ ] **코드 블록**: Python 코드가 어두운 배경의 코드 블록으로 표시되며, 구문 강조(Syntax Highlighting)가 적용됩니다.
- [ ] **키워드 색상**: `def`, `return`, `if` 등 Python 키워드가 색상으로 구분됩니다.
- [ ] **마크다운 목록**: 설명 부분이 불릿(`-`, `*`) 또는 번호(`1.`, `2.`) 리스트로 올바르게 렌더링됩니다.
- [ ] **마크다운 제목**: `#`, `##`, `###` 등 heading이 올바른 크기로 렌더링됩니다.
- [ ] **인라인 코드**: 백틱(`` ` ``)으로 감싼 인라인 코드가 배경색이 다른 스타일로 표시됩니다.
- [ ] **Agent Channel**: 여전히 "대기 중" 상태입니다.

## 5. 시나리오 4: 세션 자동 제목 생성

**목적**: 첫 메시지 기반 세션 제목 자동 생성 확인
**관련 요구사항**: F-042

### 절차

1. 시나리오 2를 진행한 직후, 좌측 사이드바와 중앙 상단 헤더를 확인합니다.

### 예상 결과

- [ ] **사이드바 반영**: 좌측 사이드바의 세션 목록에도 동일한 제목으로 세션이 표시됩니다.

## 6. 시나리오 5: 세션 생성·전환·히스토리 복원

**목적**: 다중 세션 관리, 세션 간 전환, 히스토리 DB 저장·복원 확인
**관련 요구사항**: F-023, F-040, F-041

### 절차

1. 좌측 사이드바의 **"+ 새 세션"** 버튼을 클릭합니다.
2. 메인 화면이 초기화되는지 확인합니다.
3. 새 채팅창에 `이것은 두 번째 세션입니다.`를 입력하고 전송합니다.
4. 응답을 받은 후, 사이드바에서 **첫 번째 세션**을 클릭합니다.
5. 다시 **두 번째 세션**을 클릭합니다.

### 예상 결과

- [ ] **새 세션 생성**: "새 세션" 클릭 시 User Channel이 초기화되고 "무엇이든 물어보세요" 안내가 다시 표시됩니다 (F-040).
- [ ] **Agent Channel 초기화**: Agent Channel도 "대기 중" 상태로 리셋됩니다.
- [ ] **사이드바 2개 세션**: 사이드바에 두 개의 세션이 표시됩니다 (F-040).
- [ ] **첫 번째 세션 복원**: 첫 번째 세션 클릭 시 시나리오 2~3에서 나눈 모든 대화(인사 + 피보나치 코드)가 완벽하게 복원됩니다 (F-023, F-041).
- [ ] **두 번째 세션 복원**: 두 번째 세션 클릭 시 "이것은 두 번째 세션입니다." 대화가 복원됩니다 (F-023).
- [ ] **마크다운 복원**: 복원된 대화에서 마크다운 렌더링(코드 블록, 리스트 등)이 정상 유지됩니다.
- [ ] **세션 전환 즉시성**: 세션 클릭 후 히스토리가 즉시(1초 이내) 표시됩니다.

## 8. 시나리오 7: Team 모드 전체 흐름

**목적**: 복잡한 요청 → Team 분류 → Sub-Agent 순차 실행 → 통합 응답 전체 흐름 확인
**관련 요구사항**: F-001, F-003, F-004, F-005, F-006, F-010, F-011, F-022, F-030

### 절차

1. "새 세션"을 생성합니다.
2. 다음과 같은 복잡한 요청을 입력하고 전송합니다:
   ```
   React와 FastAPI를 사용한 TODO 앱의 설계를 해줘. 아키텍처를 분석하고, 핵심 코드를 작성하고, 코드 리뷰까지 진행해줘.
   ```
3. User Channel의 상태 배지 변화를 관찰합니다.
4. Agent Channel의 메시지 표시를 관찰합니다.

### 예상 결과

**User Channel 상태 전이 (F-005, F-006)**:
- [ ] **thinking**: "분석 중..." 상태 배지가 표시됩니다.
- [ ] **delegating**: "팀 구성 중..." 상태로 변경됩니다 (F-006, F-022).
- [ ] **working**: "작업 중 (1/N)..." 형식으로 진행률이 표시됩니다. N은 Agent 수입니다.
- [ ] **working 진행**: "작업 중 (2/N)...", "작업 중 (3/N)..." 순서로 진행률이 업데이트됩니다.
- [ ] **integrating**: 모든 Agent 완료 후 "결과 통합 중..." 상태로 변경됩니다.
- [ ] **done**: 최종 통합 응답이 Reader의 회색 말풍선으로 표시됩니다 (F-004).

**Agent Channel 표시 (F-030)**:
- [ ] **대기 → 활성**: "대기 중" 상태에서 Agent 메시지 목록 표시로 전환됩니다.
- [ ] **Agent 메시지 카드**: 각 Sub-Agent의 메시지가 카드 형태로 순차 표시됩니다.
- [ ] **중간 출력**: 작업 중인 Agent의 중간 출력(부분 텍스트)이 실시간으로 업데이트됩니다 (F-030).
- [ ] **역할 다양성**: 2개 이상의 서로 다른 역할(예: Planner, Coder, Reviewer)이 표시됩니다 (F-010, F-011).
- [ ] **자동 스크롤**: 새 Agent 메시지가 나타날 때 Agent Channel이 자동 스크롤됩니다.

## 9. 시나리오 8: Agent Channel 중간 출력 및 상태 표시

**목적**: Agent 메시지 카드의 세부 요소 확인
**관련 요구사항**: F-012, F-031

### 절차

1. 시나리오 7 진행 중 Agent Channel의 메시지 카드를 관찰합니다.

### 예상 결과

- [ ] **발신자 이름**: 각 카드에 Agent 이름(예: "Agent-1")이 표시됩니다 (F-031).
- [ ] **역할 배지**: 역할 프리셋 이름(Researcher, Coder, Reviewer, Writer, Planner 중 하나)이 색상 배지로 표시됩니다 (F-031).
  - Researcher: 파란색 배지
  - Coder: 초록색 배지
  - Reviewer: 노란색 배지
  - Writer: 보라색 배지
  - Planner: 주황색 배지
- [ ] **타임스탬프**: 각 카드에 시간(예: "오후 2:15")이 표시됩니다 (F-031).
- [ ] **상태 아이콘**: 작업 중(⏳) / 완료(✅) / 에러(❌) / 취소(⏹️) 아이콘이 표시됩니다 (F-012).
- [ ] **working 커서**: 작업 중인 Agent의 본문에 깜빡이는 커서 애니메이션(pulse)이 보입니다.
- [ ] **상태 변화**: 작업이 끝나면 ⏳ → ✅로 아이콘이 변경됩니다.
