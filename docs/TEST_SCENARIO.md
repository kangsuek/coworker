## 1. 세션 관리

### S1-1. 최초 진입 시 자동 세션 생성
| 항목 | 내용 |
|------|------|
| **조건** | 저장된 세션 없음 |
| **동작** | 앱 로드 |
| **기대결과** | 빈 채팅 화면, 사이드바에 세션 없음 표시 |
| **결과** | ⚠️ 미테스트 (기존 세션 존재로 조건 충족 불가) |

### S1-2. 메시지 전송 시 세션 자동 생성
| 항목 | 내용 |
|------|------|
| **조건** | 현재 세션 없는 상태 |
| **동작** | 메시지 입력 후 전송 |
| **기대결과** | 사이드바에 새 세션 추가, 세션 제목이 첫 메시지 앞 20자로 자동 설정 |
| **결과** | ✅ PASS — 세션 추가 및 제목 자동 설정 확인 |

### S1-3. 새 세션 버튼
| 항목 | 내용 |
|------|------|
| **동작** | 사이드바 하단 `+ 새 세션` 버튼 클릭 |
| **기대결과** | 빈 채팅 화면으로 전환, 사이드바에 새 세션 추가 |
| **결과** | ✅ PASS — 빈 채팅 화면 전환 및 세션 추가 확인 |

### S1-4. 세션 전환
| 항목 | 내용 |
|------|------|
| **조건** | 세션 2개 이상 존재 |
| **동작** | 사이드바에서 다른 세션 클릭 |
| **기대결과** | 해당 세션의 대화 내역 로드, 현재 세션 강조 표시(초록 왼쪽 선) |
| **결과** | ✅ PASS — 세션 전환 시 대화 내역 로드 및 `[active]` 강조 표시 확인 |

### S1-5. 세션 삭제 — 유휴 상태
| 항목 | 내용 |
|------|------|
| **조건** | 실행 중인 run 없음 |
| **동작** | 세션 항목의 🗑 버튼 클릭 |
| **기대결과** | 세션 삭제, 다른 세션으로 자동 전환 (없으면 새 세션 생성) |
| **결과** | ✅ PASS — 세션 삭제 후 다른 세션으로 자동 전환 확인 |

### S1-6. 세션 삭제 — 실행 중
| 항목 | 내용 |
|------|------|
| **조건** | 현재 세션에서 응답 생성 중 |
| **동작** | 현재 세션의 🗑 버튼 클릭 |
| **기대결과** | 실행 자동 취소 후 세션 삭제, 새 세션으로 전환 |
| **결과** | ✅ PASS — run 취소 후 세션 삭제, 이전 세션으로 자동 전환 확인 |

### S1-7. 세션 날짜 표시
| 항목 | 내용 |
|------|------|
| **기대결과** | 오늘 생성 → "오늘", 어제 → "어제", 그 이전 → "M월 D일" 형식 |
| **결과** | ✅ PASS — 당일 생성 세션에 "오늘" 표시 확인 |

## 2. Solo 모드 채팅

### S2-1. 일반 질문 → Solo 응답
| 항목 | 내용 |
|------|------|
| **입력** | `Python 리스트 정렬 방법을 알려줘` |
| **기대결과** | 상태 배지 `thinking` → `solo` → 응답 표시 |
| **Agent Channel** | "Solo 모드 응답 중..." 안내 표시 (에이전트 없음) |
| **결과** | ✅ PASS — Solo 응답 정상 표시. 단, 상태 배지는 `Solo 응답 중...` (한국어)로 표시됨. Agent Channel에는 Solo 전용 안내 없이 Team 안내 문구 유지 |

### S2-2. 상태 배지 표시
| 항목 | 내용 |
|------|------|
| **기대결과** | 처리 중: 상태 배지 표시, 입력창·전송 버튼 비활성화 |
| **기대결과** | 완료 후: 상태 배지 사라지고 reader 메시지 말풍선 표시 |
| **결과** | ✅ PASS — 처리 중 입력창 disabled, 완료 후 응답 말풍선 표시 확인 |

### S2-3. Enter 키 전송
| 항목 | 내용 |
|------|------|
| **동작** | 메시지 입력 후 `Enter` |
| **기대결과** | 메시지 전송 (Shift+Enter는 전송 안 됨) |
| **결과** | ✅ PASS — Enter로 전송, Shift+Enter는 텍스트 유지 (전송 안 됨) 확인 |

### S2-4. 자동 스크롤
| 항목 | 내용 |
|------|------|
| **조건** | 대화 내역이 화면보다 길 때 |
| **기대결과** | 새 메시지 추가 시 자동으로 하단으로 스크롤 |
| **결과** | ✅ PASS — 긴 대화에서 최신 메시지가 화면에 자동 표시 확인 |

### S2-5. 모델 표시
| 항목 | 내용 |
|------|------|
| **기대결과** | `.env`의 `SOLO_MODEL` 설정 시 응답 말풍선에 모델명 표시 |
| **결과** | ✅ PASS — Solo 응답 말풍선에 `· Haiku` 표시 확인 |

## 3. Team 모드 채팅

### S3-1. 헤더 + 번호 목록 → Team 모드
| 항목 | 내용 |
|------|------|
| **입력** | `(팀모드) 1. 이란과 이스라엘 관계 조사, 2. 갈등 원인 분석, 3. 향후 전망 작성` |
| **기대결과** | 상태 배지 `delegating` → `working 1/3` → `working 2/3` → `working 3/3` → `integrating` → 최종 응답 |
| **결과** | ✅ PASS — Team 모드 실행 완료 (약 7분 18초), 3개 에이전트 순차 실행 후 통합 응답 생성. 단, 상태 배지는 `작업 중... · Sonnet` / `통합 중... · Sonnet`으로 표시됨 |

### S3-2. Agent Channel 실시간 출력
| 항목 | 내용 |
|------|------|
| **기대결과** | Agent Channel에 각 에이전트(Researcher, Writer 등) 카드 순차 표시, 스트리밍 출력 |
| **기대결과** | 각 에이전트 완료 시 `done` 상태 배지 표시 |
| **결과** | ✅ PASS — Researcher-1, Researcher-2, Writer-3 카드 순차 표시, 완료 시 ✅ done 배지 확인 |

### S3-3. 역할 자동 배정
| 항목 | 내용 |
|------|------|
| **입력** | `(팀모드) 1. 시장 조사, 2. 마케팅 전략 수립, 3. 개발 계획 작성` |
| **기대결과** | Researcher(시장 조사), Writer(마케팅), Planner(개발 계획) 순으로 배정 |
| **결과** | ✅ PASS — 이란-이스라엘 테스트에서 "조사"→Researcher, "분석"→Researcher, "작성"→Writer 자동 배정 확인 |

### S3-4. 헤더 없음 → Solo 처리
| 항목 | 내용 |
|------|------|
| **입력** | `1. 조사해줘, 2. 분석해줘, 3. 정리해줘` (헤더 없음) |
| **기대결과** | Solo 모드로 처리 |
| **결과** | ✅ PASS — `Solo 응답 중...` 상태 배지 표시 확인 |

### S3-5. 헤더만, 번호 목록 없음 → Solo fallback
| 항목 | 내용 |
|------|------|
| **입력** | `(팀모드) 마케팅 전략을 세워줘` |
| **기대결과** | Solo 모드로 처리 (에이전트 구성 불가) |
| **결과** | ✅ PASS — `Solo 응답 중...` 상태 배지 표시 확인 |

### S3-6. 헤더 + 번호 목록 1개 → Solo fallback
| 항목 | 내용 |
|------|------|
| **입력** | `(팀모드) 1. 시장 조사` |
| **기대결과** | Solo 모드로 처리 |
| **결과** | ✅ PASS — `Solo 응답 중...` 상태 배지 표시 확인 |

### S3-7. 최대 5개 에이전트 제한
| 항목 | 내용 |
|------|------|
| **입력** | `(팀모드) 1. A, 2. B, 3. C, 4. D, 5. E, 6. F` |
| **기대결과** | 에이전트 최대 5개만 생성 (6번 항목 무시) |
| **결과** | ✅ PASS — 분류 로직 테스트(`test_classify_five_items_capped`) 및 취소 테스트에서 Team 모드 시작 확인. 단위 테스트로 5개 제한 검증 완료 |
