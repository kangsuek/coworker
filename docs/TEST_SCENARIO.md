# Coworker - Sprint 4까지의 수동 테스트 시나리오

본 문서는 Phase 1의 Sprint 4까지 개발된 **Solo 모드 및 UI/세션 관리** 기능이 정상적으로 동작하는지 수동으로 검증하기 위한 시나리오입니다.

## 1. 테스트 환경 실행 방법

프로젝트 루트 디렉토리(`/Users/kangsuek/pythonProject/coworker`)에서 터미널을 열고 다음 명령어들을 실행합니다.

```bash
# 1. 의존성 설치 (최초 1회 혹은 패키지 변경 시)
just setup

# 2. 데이터베이스 초기화 및 마이그레이션 (필수)
just migrate

# 3. 개발 서버 실행 (백엔드와 프론트엔드가 동시에 실행됨)
just dev
```

> **참고**: `just dev`를 실행하면 프론트엔드는 `http://localhost:5173`, 백엔드는 `http://localhost:8000`에서 실행됩니다. 터미널 하나를 띄워두고 백그라운드로 돌거나 각각 탭을 열어 실행할 수도 있습니다.
> 웹 브라우저를 열어 **`http://localhost:5173`** 에 접속합니다.

## 2. 테스트 시나리오

### 시나리오 1: 단순 인사 및 Solo 모드 기본 응답
- **목적**: 입력 전송, 상태 변화(상태 배지), Claude CLI를 통한 응답 및 자동 스크롤 확인
- **절차**:
  1. 화면 우측 메인 채팅창(User Channel)의 입력 칸에 `안녕하세요! 당신은 누구인가요?` 라고 입력합니다.
  2. `Enter` 키를 누르거나 `전송` 버튼을 클릭합니다.
  3. **예상 결과**:
     - 입력창이 비활성화되고 메시지가 파란색 말풍선으로 화면에 나타납니다.
     - 말풍선 아래에 `분석 중... (thinking)` -> `Solo 응답 중... (solo)` 순서로 상태 배지가 애니메이션과 함께 변경됩니다.
     - 약 수 초 내에 상태가 `완료 (done)`로 바뀌며 Reader Agent의 답변이 회색 말풍선으로 나타납니다.
     - 답변이 나타날 때 최하단으로 자동 스크롤됩니다.

### 시나리오 2: 세션 자동 제목 생성 및 좌측 사이드바 갱신
- **목적**: 첫 번째 메시지 전송 시 사이드바에 새로운 세션이 생성되고, 메시지 내용에 기반해 제목이 자동 생성되는지 확인
- **절차**:
  1. (시나리오 1을 진행한 직후) 좌측 사이드바를 확인합니다.
  2. **예상 결과**:
     - 상단 헤더에 `새 대화`라고 표시되던 영역이 `안녕하세요! 당신은 누...` 와 같이 첫 질문의 앞 30자로 자동 변경됩니다.
     - 좌측 사이드바의 세션 목록에도 해당 제목으로 세션이 생성 및 하이라이트 되어 있습니다.

### 시나리오 3: 마크다운 및 코드 구문 강조(Syntax Highlighting) 확인
- **목적**: 프론트엔드의 `react-markdown`과 `rehype-highlight`가 정상 동작하는지 확인
- **절차**:
  1. 동일한 채팅창에 `파이썬으로 1부터 10까지 더하는 함수를 작성해줘. 그리고 코드에 대해 마크다운 리스트 형식으로 간단히 설명해줘.` 라고 입력하고 전송합니다.
  2. **예상 결과**:
     - 응답 내용 중 Python 코드가 별도의 어두운 배경/코드 블록 형태로 구문 강조(Syntax Highlighting)되어 나타납니다.
     - 설명 부분이 `1.`, `2.` 혹은 `-`, `*` 등 올바른 리스트(목록) UI로 렌더링 됩니다.
     - 우측 `Agent Channel` 패널은 여전히 🤖 아이콘과 함께 "대기 중" 상태를 유지합니다. (요청이 단순 코딩/질문이므로 Solo 모드로 라우팅 됨)

### 시나리오 4: 세션 전환 및 히스토리 복원
- **목적**: 다중 세션 관리 및 이전 대화 내역(DB) 로드 확인
- **절차**:
  1. 좌측 사이드바 하단의 **`+ 새 세션`** 버튼을 클릭합니다.
  2. 메인 화면이 초기화되고 메시지 목록이 비워지는 것을 확인합니다.
  3. 새 대화창에 `이것은 두 번째 세션입니다.` 라고 입력하고 전송하여 응답을 받습니다.
  4. 다시 좌측 사이드바에서 **첫 번째로 생성했던 세션**을 클릭합니다.
  5. **예상 결과**:
     - 클릭 즉시 화면이 첫 번째 세션으로 전환됩니다.
     - 이전에 나누었던 인사, 파이썬 코드 요청 내역 2세트가 모두 완벽하게 화면에 복원되어 표시됩니다.

### 시나리오 5: 사이드바 토글 (접기/펼치기)
- **목적**: 레이아웃 반응성 확인
- **절차**:
  1. 중앙 채팅창의 좌측 상단 헤더에 있는 **`☰`** (햄버거 메뉴) 버튼을 클릭합니다.
  2. **예상 결과**:
     - 좌측 세션 사이드바가 부드럽게 사라지며 중앙 채팅창이 전체 화면으로 넓어집니다.
     - 다시 누르면 사이드바가 나타납니다.
