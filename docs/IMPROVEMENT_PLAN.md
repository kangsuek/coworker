# 시스템 기능 개선 상세 계획 (Improvement Plan)

`docs/code_review.md`에서 제기된 문제점들을 해결하고 시스템의 성능과 사용자 경험을 향상시키기 위한 단계별 로드맵입니다.

## Phase 1: 컨텍스트 관리 강화 (기억력 개선)
**우선순위: 높음 | 목표: Team 모드의 문맥 인지 능력 확보**

### 1.1. Team 모드 히스토리 전달 로직 개선
- **대상**: `backend/app/agents/reader.py`
- **변경 사항**: 
    - `_team_execute` 메서드 내에서 세션의 최근 대화 이력을 로드하도록 수정.
    - `SubAgent.execute` 호출 시 `context` 파라미터에 사용자 메시지뿐만 아니라 이전 대화의 요약본을 포함.
- **기대 효과**: "아까 말한 설정대로 해줘"와 같은 지시를 Team 모드에서도 이해 가능.

### 1.2. 지능형 컨텍스트 요약 (Summarization)
- **변경 사항**:
    - 대화 내용이 3,000자를 초과할 경우 자동으로 배경 맥락을 요약하는 `_summarize_context` 기능을 개선.
    - 단순 삭제가 아닌 핵심 정보(코드 구조, 설정값, 주요 의사결정) 위주의 요약 수행.

---

## Phase 2: 실행 속도 최적화 (성능 개선)
**우선순위: 높음 | 목표: 응답 대기 시간 50% 단축**

### 2.1. 에이전트 병렬 실행 (Parallel Execution)
- **변경 사항**:
    - 의존 관계가 없는 태스크를 분류 단계에서 식별.
    - `ReaderAgent`에서 `for` 루프 대신 `asyncio.gather`를 사용하여 독립적인 에이전트 태스크를 동시 실행.
- **제약 사항**: LLM API의 Rate Limit 및 로컬 리소스(CLI Lock)를 고려하여 `asyncio.Semaphore`로 동시 실행 개수(예: 최대 3개) 제한.

### 2.2. 프론트엔드 폴링 방식 개선 (Streaming)
- **변경 사항**:
    - 현재 2초 간격의 HTTP 폴링을 Server-Sent Events (SSE) 방식으로 전환.
    - `backend/app/routers/chat.py`에 스트리밍 엔드포인트 추가.
- **기대 효과**: 응답 완료 후 대기 시간(최대 2초) 제거 및 실시간 타이핑 효과 제공.

---

## Phase 3: 신뢰성 및 안정성 강화
**우선순위: 보통 | 목표: 오류 복구 및 품질 보장**

### 3.1. 에이전트 자가 검토 (Self-Review) 루프
- **변경 사항**:
    - `Reviewer` 에이전트의 역할을 강화하여, 다른 에이전트의 결과가 요청 사항을 충족하지 못할 경우 자동으로 재실행(Retry)을 트리거하는 로직 구현.
- **기대 효과**: 한 번의 실행으로 더 완성도 높은 결과물 도출.

### 3.2. 상태 관리 정교화
- **변경 사항**:
    - 에이전트 실행 중 오류 발생 시, 전체 Run을 `error`로 종료하기보다 해당 태스크만 재시도하거나 대체 에이전트를 투입하는 Fallback 메커니즘 검토.

---

## Phase 4: 확장 기능 (미래 과제)
- **Vector DB 도입**: 대규모 프로젝트 코드를 분석하기 위한 RAG(Retrieval-Augmented Generation) 기반 검색 시스템 구축.
- **멀티 세션 오케스트레이션**: 여러 세션 간에 지식을 공유하거나 협업하는 기능.
