백엔드
session_service.delete_session(session_id)
해당 세션의 Run, AgentMessage, UserMessage를 삭제한 뒤 세션 삭제.
세션이 없으면 False 반환.
DELETE /api/sessions/{session_id}
204 No Content 반환.
404: 해당 세션 없음.
테스트

프론트엔드
api.deleteSession(id)
DELETE /api/sessions/{id} 호출.
useSession.deleteSession(id)
삭제 후 목록 갱신.
삭제한 세션이 현재 세션이면: 남은 세션이 있으면 첫 번째로 전환, 없으면 새 세션 생성.
SessionList

클릭 시 onDeleteSession(sess.id) 호출, 클릭 이벤트는 상위로 전파하지 않음(세션 전환 방지).
onDeleteSession prop 추가.
UI

삭제 버튼: 호버 시 빨간색 톤, 툴팁 "세션 삭제".
